### 🙋‍♂️ About Me

**프론트엔드 개발자를 꿈꾸는 대학생 khy1121입니다.**

단순히 동작하는 코드를 넘어, *더 나은 구조*를 고민하는 개발자가 되고 싶습니다.
`SwingFpsGame` 프로젝트에서는 완성된 FPS 게임을 그대로 두지 않고, 불필요한 로직과 중복 코드를 걷어내는 리팩토링을 진행하며 "동작하는 코드"와 "좋은 코드"의 차이를 직접 체감했습니다. 이 경험은 이후 프로젝트에서 컴포넌트 구조와 상태 관리를 먼저 고민하는 습관으로 이어졌습니다.

`LookMate` 프로젝트에서는 가상 피팅(Virtual Try-On), AI 옷장 등 여러 기능이 얽힌 요구사항을 TypeScript 기반으로 설계하며, 기능 단위를 어떻게 나누고 타입으로 안전하게 연결할지 문제를 정의하고 해결하는 과정을 경험했습니다.

- 🎯 **관심 분야**: 사용자 경험을 고려한 프론트엔드 개발, React 기반 웹/앱 서비스
- 🧩 **문제 해결 방식**: 일단 동작하게 만든 뒤, "왜 이렇게 짰는가"를 스스로 설명할 수 있을 때까지 구조를 다듬습니다
- 🌱 **성장 중인 부분**: React와 TypeScript를 중심으로 컴포넌트 설계, 상태 관리, 협업 워크플로우
- 📫 **연락처**: (원하시면 이메일/블로그/포트폴리오 링크 추가)

> "빠르게 만드는 것보다, 왜 이렇게 만들었는지 설명할 수 있는 코드를 짜자."
